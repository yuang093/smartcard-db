# 專案概述 (Project Overview)
本專案旨在開發一個支援多使用者 (Multi-tenant) 的智慧型名片管理 Web 系統。使用者可上傳名片正反面照片，系統將透過 AI 解析圖片內容並建立結構化資料庫，同時極力優化手機端的操作體驗。

## 系統架構設計
- **前端 (Frontend)**: Next.js + Tailwind CSS (RWD 響應式設計，手機端優先)。
- **後端 (Backend)**: FastAPI (Python) 或 Next.js API Routes。
- **資料庫 (Database)**: PostgreSQL (以 `user_id` 嚴格隔離不同使用者的資料)。
- **AI/OCR 引擎**: MiniMax 視覺大模型 (或 OpenClaw 支援之其他 VLM)，負責解析名片圖檔並回傳 JSON 格式資料，以最佳化 API 成本。
- **部署環境**: Mac mini 內的 Docker 容器。
- **網域配置**: `card.yuang093.cc`。
- **版本控制與備份**: 原始碼與設定檔自動備份至 GitHub。

## 核心功能需求
1. **多使用者與身分驗證 (Auth)**: 
   - 獨立的登入/註冊系統與管理員帳號管理頁面。
   - 嚴格的資料庫隔離，各帳號資料互不干涉。
2. **名片上傳與 AI 辨識**: 
   - 支援手機端相機拍照直傳或圖片上傳 (正反兩面)。
   - 透過 MiniMax AI 萃取：姓名、公司、職稱、電話、手機、Email、地址。
   - 辨識結果需具備人工校對與修改介面。
3. **資料庫管理與搜尋**: 
   - **標籤系統 (Tags)**: 支援 AI 自動分類標籤，且使用者必須能**手動新增、編輯、刪除**標籤。
   - **重複偵測**: 上傳或建立時，自動比對現有資料庫 (依據姓名、電話或 Email)，提示是否有重複聯絡人。
   - 全域快速搜尋 (姓名、公司、標籤)。
4. **資料匯出與同步**:
   - 列表匯出成 Excel (.xlsx) 格式。
   - **一鍵匯出通訊錄**: 單張名片支援生成標準 vCard (.vcf) 檔案或 QR Code，確保相容匯入 Google 通訊錄與 iPhone 手機通訊錄。
